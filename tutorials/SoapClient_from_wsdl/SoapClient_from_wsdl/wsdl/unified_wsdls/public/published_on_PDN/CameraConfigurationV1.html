CameraConfiguration - Web Service


$Id: CameraConfigurationV1.wsdl 595 2011-05-17 17:49:38Z $

This service allows a client to:

  • query a camera device's current configuration state
  • update a camera device's configuration state
  • reset a camera device's configuration state to factory defaults

Pelco IP-based devices use the newer StreamConfiguration service instead, except in cases where contrast, saturation, and brightness settings need to be modified.
Target Namespace:
urn:schemas-pelco-com:service:CameraConfiguration:1
[toc]

Operations

    1. GetConfiguration

      Returns a given camera's current configuration.
      SOAP action:
      urn:schemas-pelco-com:service:CameraConfiguration:1#GetConfiguration
      Input:
      [GetConfiguration]
      Request to return the values for the currently active mode of the camera.
      Output:
      [GetConfigurationResponse]
      Returns the values for the currently active mode of the camera
      • cameraConfig [CameraConfig]
        • videoproperties - optional; [VideoProperties]
          • brightness - optional; [MVElement]
            The amount of light in the frame.
            • mode [int]
              0 for manual, 1 for automatic
            • value [int]
              0 - 100, 50 is a typical value. This is only used if the mode value is 0 (manual).
          • contrast - optional; [MVElement]
            Contrast is defined as the separation between the darkest and brightest areas of the frame. As contrast is increased, the separation between dark and bright is increased, making shadows darker and highlights brighter. As contrast is decreased, the shadows increase and the highlights are decreased to make the values closer to one another.
            • mode [int]
              0 for manual, 1 for automatic
            • value [int]
              0 - 100, 50 is a typical value. This is only used if the mode value is 0 (manual).
          • hue - optional; [MVElement]
            A specific color range classification.
            • mode [int]
              0 for manual, 1 for automatic
            • value [int]
              0 - 100, 50 is a typical value. This is only used if the mode value is 0 (manual).
          • saturation - optional; [MVElement]
            The intensity of a hue.
            • mode [int]
              0 for manual, 1 for automatic
            • value [int]
              0 - 100, 50 is a typical value. This is only used if the mode value is 0 (manual).
          • filterLevel [int]

            The noise filter level to be applied. Must be one of the following values:

            • 0 for LOW
            • 1 for MEDIUM
            • 2 for HIGH 0 is a typical value.

        • videoQuality - optional;
          DEPRECATED. Available only for backwards compatibility.
          Determines the quality of video recordings.
        • coaxMode - optional; [int]
          DEPRECATED. Available only for backwards compatibility.

          The coaxitron mode to be used to control the camera. Currently acceptable values are:

          • 0 for DISABLED
          • 1 for STANDARD
          • 2 for EXTENDED

        • controlMode - optional; [ConfigurableInt]

          Describes the type of camera control. This is restricted to the following values:

          • 0 for DISABLED
          • 1 for COAXITRON (BNC COAX)
          • 2 for EXCOAXITRON (BNC COAX)
          • 3 for DPROTOCOL (2 wire RS485 or 4 wire RS422)
          • 4 for PPROTOCOL (deprecated)

        • cameraAddress [int]
          An index of the camera's location within a group of cameras.
        • cameraType - optional; [ConfigurableInt]
          DEPRECATED. Available only for backwards compatibility.

          Describes the type of camera.

          • configurable - set to 1 (true) if configurable, 0 (false) otherwise. This is restricted to the following values:
            • 0 for FIXED - unmovable camera
            • 1 for PANTILT - a movable camera that has a greater range of motion
            • 2 for DOME - a camera that utlizes a dome enclosure

        • distinguishers - optional; [Distinguishers]
          • viewobject - optional; [int]
            For cameras supporting the view object command set this value to 1, otherwise set it to 0 or omit the viewobject sub element entirely.
          • dome - optional; [int]
            For dome cameras set this value to 1, otherwise set it to 0 or omit the dome sub element entirely.
          • ptz - optional; [int]
            For ptz cameras set this value to 1, otherwise set it to 0 or omit the ptz sub element entirely.
          • fixed - optional; [int]
            For fixed cameras set this value to 1, otherwise set it to 0 or omit the fixed sub element entirely.
        • motionDetectionEnable [int]

          Enable or disable motion detection service. This is restricted to the following values:

          • -1 for unsupported
          • 0 for disabled
          • 1 for enabled

        • motionAlarmDwellSecs - optional; [int]
          The amount of time the motion alarm should stay “high” after it is activated.
        • audioEnable [int]

          Determines the camera's current audio status. Must be one of the following values:

          • -1 for TS_UNSUPPORTED
          • 0 for TS_DISABLED
          • 1 for TS_ENABLED

        • termination [int]
          Set to 1 (true) to enable termination, 0 (false) otherwise.
        • audioHardwareInput - optional; [int]

          Indicates what source to use for audio input. This setting is primarily used by encoders that do not have a DIP switch that controls this setting. Must be set to one of the following:

          • -1 for TS_AUDIO_HW_INPUT_UNSUPPORTED: If the input cannot be configured through software then it this should be set to unsupported.
          • 0 for TS_AUDIO_HW_INPUT_LINE_IN: To use the line in for input.
          • 1 for TS_AUDIO_HW_INPUT_MIC: To use the mic port as the input.

        • ttl [int]

          The time to live of the IP datagrams of RTPpackets.

          • Valid range is 1 - 255.
          • 255 is the average value used
          • TTL value gets subtracted by 1 for each router it goes through. When zero is reached, the package is discarded.

        • packetsize [int]
          The size of each network packet.

            • videosize [int]

              The resolution of the camera's generated video stream. Can be one of the following values:

              • 0 for QCIF NTSC:176x112 PAL:176x144
              • 1 for CIF NTSC:352×240 PAL:352x288
              • 2 for 2CIF NTSC:704x240 PAL:704x288
              • 3 for 4CIF NTSC:704x480 PAL:704x576
              • 4 for D1 NTSC:720x480 PAL:720x576

            • tvformat [int]

              The TV Format of the camera's generated video stream. Must be one of the following values:

              • 0 for NTSC
              • 1 for PAL

            • pruneStoredVideo - optional; [int]
              This configuration values is stored on the camera but has no effect on it. The value is read by a NSD/NVR which can then change its recording behavior based on the value of this setting.
            • gop [double]

            • format - optional, nillable; [string]

              The file format of the video stream. Must be one of the following values:

              • Mpeg-4
              • H.264

            • videosize [int]

              The resolution of the camera's generated video stream. Can be one of the following values:

              • 0 for QCIF NTSC:176x112 PAL:176x144
              • 1 for CIF NTSC:352×240 PAL:352x288
              • 2 for 2CIF NTSC:704x240 PAL:704x288
              • 3 for 4CIF NTSC:704x480 PAL:704x576
              • 4 for D1 NTSC:720x480 PAL:720x576

            • tvformat [int]

              The TV Format of the camera's generated video stream. Must be one of the following values:

              • 0 for NTSC
              • 1 for PAL

            • pruneStoredVideo - optional; [int]
              This configuration values is stored on the camera but has no effect on it. The value is read by a NSD/NVR which can then change its recording behavior based on the value of this setting.
            • gop [double]
              The number of frames in one GOP (Group of Pictures). The typical value ranges from 4-20 frames.

          Set to 1 (true) to use the configuration settings found on camera itself; otherwise set to 0 (false) to use global settings found on the local Endura System Manager.

    2. ResetConfiguration

      Resets a camera's configuration to factory defaults.

      It is not recommended to use this action for PTZ cameras, since the factory default is to disable PTZ functionality.
      SOAP action:
      urn:schemas-pelco-com:service:CameraConfiguration:1#ResetConfiguration
      Input:
      [ResetConfiguration]
      Represents the data for a ResetConfiguration request.
      Output:
      [ResetConfigurationResponse]
      Represents the data for a ResetConfiguration response.

    3. SetConfiguration

      Sets the current camera configuration to the configuration indicated by cameraConfig.

      Every time you attempt to modify the camera's configuration, you must send a full camera config parameter. (Preferably, one you just retreived from GetConfiguration and modified.) Failure to do so will result in settings (specifically the ones that were not initialized in the CameraConfig instance) being reverted to factory defaults.

      Failure to use known valid settings may cause unexpected camera behavior.

      The configuration changes apply to the currently active camera mode (e.g., normal vs. low-light, etc.).

      SOAP action:
      urn:schemas-pelco-com:service:CameraConfiguration:1#SetConfiguration
      Input:
      [SetConfiguration]
      Applies a specific set of configurations to the currently active mode of the camera.
