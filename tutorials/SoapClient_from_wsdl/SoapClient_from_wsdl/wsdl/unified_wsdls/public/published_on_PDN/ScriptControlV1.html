ScriptControl - Web Service


Target Namespace:
urn:schemas-pelco-com:service:ScriptControl:1
Description:
This service allows clients to control scripts, presets, and patterns.
Transport protocol:
SOAP over HTTP
Operations:
  1. BeginScriptDetail
  2. CancelScriptDetail
  3. DeleteScriptDetail
  4. EndScriptDetail
  5. ExecuteScriptDetail
  6. HaltScriptDetail
  7. QueryScriptsDetail

Operations

    1. BeginScript
      Description:
      Causes the service instance to begin "recording" or "programming" the script. The name of the script is set in scriptName.
      Only D-Protocol devices behave properly with BeginScript. Currently there is a known issue with EXCOAXITRON: If the preset name does not end in a number that starts with 1 OR if other presets already exist and the the preset name does not end in a consecutive number continuing from the last preset number, then the new preset will fail (for example, if the last preset name ended with 3, then the next preset name must end with 4). Also, the BeginScript action only applies to devices that support recording and programming. For example, dome positioning systems have a feature called pattern recording, which allows the user to record a specific set of movement commands into a movement pattern (this is also known as a script). The BeginScript action starts the recording process.
      SOAP action:
      urn:schemas-pelco-com:service:ScriptControl:1#BeginScript
      Input:
      [BeginScript]
      A request to the BeginScript action.
      • scriptName [string]
        The name of the preset or pattern to begin 'recording'.
      Output:
      [BeginScriptResponse]
      A response to a BeginScript request.
    2. CancelScript
      Description:

      SOAP action:
      urn:schemas-pelco-com:service:ScriptControl:1#CancelScript

      [CancelScript]
      A request to the CancelScript action.
      • scriptName [string]
        The name of the script to cancel 'recording'.
      Output:
      [CancelScriptResponse]
      A response to a CancelScript request.
    3. DeleteScript
      Description:
      Causes the service instance to delete from its list of scripts the script indicated by scriptName.

      urn:schemas-pelco-com:service:ScriptControl:1#DeleteScript
      Input:
      [DeleteScript]
      A request to the DeleteScript action.
      • scriptName [string]
        The name of the script to delete.
      Output:
      [DeleteScriptResponse]
      A response to a DeleteScript request.
    4. EndScript
      Description:
      Causes the service instance to end the "recording" or "programming" of a specific script and save it using a given scriptName.
      SOAP action:
      urn:schemas-pelco-com:service:ScriptControl:1#EndScript
      Input:
      [EndScript]
      A request to the EndScript action.
      • scriptName [string]
        The name of the preset or pattern to end 'recording'.
      Output:
      [EndScriptResponse]
      A response to a EndScript request.
    5. ExecuteScript
      Description:
      Causes the locally stored script indicated by scriptName to be executed.
      SOAP action:
      urn:schemas-pelco-com:service:ScriptControl:1#ExecuteScript
      Input:
      [ExecuteScript]
      A request to the ExecuteScript action.
      • scriptName [string]
        The name of the script to run.
      Output:
      [ExecuteScriptResponse]
      A response to a ExecuteScript request.
      • pid [int]
        A process ID related to the currently running script.
    6. HaltScript
      Description:
      Causes an instance of a script indicated by pid to be stopped, if currently running.
      SOAP action:
      urn:schemas-pelco-com:service:ScriptControl:1#HaltScript
      Input:
      [HaltScript]
      A request to the HaltScript action.
      • scriptName - optional; [string]
        The name of the script to stop running.
      • pid - optional; [int]
        A process ID related to the currently running script.
      Output:
      [HaltScriptResponse]
      A response to a HaltScript request.
    7. QueryScripts
      Description:
      Retrieves a list of names of locally stored scripts from the service instance.
      SOAP action:
      urn:schemas-pelco-com:service:ScriptControl:1#QueryScripts
      Input:
      [QueryScripts]
      A request to the QueryScripts action.
      • scriptQuery [ScriptQuery] Defines a query to return a list of scripts.
        • name - optional; [string]
          The script's name. It has a 40 max character length. e.g. scriptname1.xml
        • description - optional; [string]
          Information regarding the script in question. It has a 250 max character length.
        • id - optional; [int]
          The script"s unique identifier.
        • owner - optional; [int]
          The script's owner.
        • executing - optional; [boolean]
          Set to 1 (true) for scripts that are currently running. Set to 0 (false) otherwise.
        • queryId - optional; [string]
          A unique identifier for the query related to these results. This is in UUID format.
        • nextEntry - optional; [int]
          The index that marks where the next page of results begins, for results that span more than one page.
        • maxEntries - optional; [int]
          The maximum number of results to return (total returned result set).
      Output:
      [QueryScriptsResponse]
      A response to a QueryScripts request.
      • scriptList [ScriptList]
        • scripts - optional; [Scripts] A list of scripts.
          • script - optional, unbounded; [Script] Contains data related to a script, pattern, or preset.
            • name [string]
              The script's name. It has a 40 max character length. e.g. scriptname1.xml
            • description - optional; [string]
              Information regarding the script in question. It has a 250 max character length.
            • id [int]
              The script"s unique identifier.
            • owner [string]
              The script's owner.
            • pids - optional; [Pids] A list of process IDs.
              • pid - optional, unbounded; [string]
                An identifier for a process running a script.
        • queryId - optional;
          A unique identifier for the query related to these results. This is in UUID format.
        • nextEntry - optional; [int]
          An index bookmarking the next entry that signifies the next page of results.
        • maxEntries - optional; [int]
          The maximum number of results to return.
        • maxPresets - optional; [int]
          The maximum number of presets, saved device states, supported by the device.
        • maxPatterns - optional; [int]
          The maximum number of patterns, a group of commands run over a time frame that can be saved and played back, supported by the device.
        • totalEntries - optional; [int]
          The total number of entries that the device can support.
        • resultURI - optional; [string]
          The URL from which the search results may be downloaded. This may be used in conjuntion with entry paging to improve performance.
