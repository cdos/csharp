VideoOutput - Web Service


$Id: VideoOutputV1.wsdl 637 2011-10-05 17:06:57Z $
This services allows clients to query existing streams and active sessions along with their parameters. Other uses for this service include initiating a stream, terminating an existing stream and refreshing sessions.
Target Namespace:
urn:schemas-pelco-com:service:VideoOutput:1
[toc]

Operations

    1. Connect

      Initiates a streaming session containing the video data indicated by streamParameters.
      SOAP action:
      urn:schemas-pelco-com:service:VideoOutput:1#Connect
      Input:
      [Connect]
      Represents a request to the Connect action
      • streamParameters [StreamParameters]
        The stream parameters used for connecting
        Only the following data needs to be set with the Connect request (for the streamParameters parameter):
        • transportUrl - The URL to which the selected video data will be transmitted.By default, video streams are multicast unless this value is filled.
        • The device port is optional; the server response will include this if it is not specified in the request.
        • refreshInterval - The interval in seconds to be used in the RefreshSession action in the urn:schemas-pelcocom: StreamControl service indicated by controlURI. If the camera does not receive any calls to the RefreshSession action within the specified interval, the video stream session identified by sessionId will be disconnected. Units are in seconds. The minimum value is 30 and the maximum value is 900.
        • displayName - optional; [string]
          The Stream's friendly name
        • eventType - optional; [string]
          Recording or event type that caused the stream to be recorded.The following are the valid values
          • Scheduled
          • Motion
          • Alarm
          • Manual
          Primarily only returned by NVR/NSM queries.
        • preAlarmDurationInManualRecord [int]
          The amount of time , in seconds prior to the alarm time that the recording should begin
          • 30 (seconds) is common
          • Used when a manual recording is initiated
          • The feature is based on the premise that the recording device has cached some amount of video prior to the current time
          This identifier is a misnomer: its functionality has little to do with alarms themselves.
        • streamSession [StreamSession]
          The Session parameters for the Stream
          • sourceURI - optional; [anyURI]
            Represents URI of orginal source of video data.
          • outputURI - optional; [anyURI]
            Represents the URI to which video data will be streamed.
          • controlURI - optional; [anyURI]
            Represents the URI using which the stream could be controlled.
          • transportURL - optional; [anyURI]
            The URL to which selected video data will be transmitted.
          • streamState - optional; [string]
            Represents current transmission state of the stream Possible states are
            • Play
            • Pause
            • Reverse
            • Seek
          • displayName - optional; [string]
            Friendly name of the session
          • sessionId - optional; [string]
            Unique Idenitifier for the Session
          • iFrameOnly [int]
            Represents whether to send IFrames only. Possible values are
            • 1 - True, only IFrames would be transmitted
            • 0 - False, All frames would be transmitted
          • secsPerIFrame - optional; [int]
            The number of seconds to display each I-Frame
          • startTime - optional; [string]
            Primarily used for playback and represents the start time of the interval being played back.
            • The format of start time is dateTime.tz. ex 1988-04-07T18:39:09-08:00
            • UTC timezone
            • NOW is also valid start time
            • ex 2007-05-02T18:48:46.017
          • endTime - optional; [string]
            Primarily used for playback and represents the end time of the interval being played back.
            • The format of end time is dateTime.tz. ex 1988-04-07T18:39:09-08:00
            • UTC timezone
            • INFINITE is also valid end time, signifying no end time
            • ex 2007-05-02T18:48:46.017
          • clients - optional; [unsignedInt]
            Represents the number of clients currently attached to this session
          • h264StreamType - optional; [H264StreamType]
            Represents the type for streaming H264 from a recoder. Valid types are
            • BYTE_STREAM
            • RFC_3984
            The H264 Stream comming from Sarix Cameras is always RFC 3984 based and Sarix cameras do not support byte stream format. By default the H264 stream type from a recoder is byte stream format.
            - BYTE_STREAM: Byte stream format
            - RFC_3984: RFC 3984 format
          • refreshInterval [unsignedInt]
            The interval in seconds to be used in the RefreshSession action in the StreamControl service indicated by controlURI. If the camera does not receive any calls to the RefreshSession action within the specified interval, the video stream session identified by sessionId will be disconnected. Valid values range from 30 - 900 seconds (15 minutes)
          • transportUrlUseSourceAddress - optional; [int]
            If specified and non-zero, when requesting a unicast stream, tells the device to replace the ip address specified in transportURL with the originating ip address of the request
        • compatability - optional; [Compatibility]
          • resolution [Resolution]
        • videoQuality - optional; [VideoQuality]
          The Video Quality parameters such as framerate,bitrate, etc
          • framerate [decimal]
            Framerate of the stream, in fps. ex 30 for NTSC, 25 for PAL
            Our devices are NTSC centric. To cater to PAL settings, simply convert the PAL value to an NTSC value: multiply the PAL value by 30/25.
          • bitrate [int]
            Bitrate of the stream, in bps.
            Our devices are NTSC centric. To cater to PAL settings, simply convert the PAL value to an NTSC value: multiply the PAL value by 30/25.
          • format - optional; [string]
            Format of the stream. Can be one of the following:
            • Mpeg-4
            • H.264
            The H.264 value is currently only valid for our Sarix product line.
          • videosize [unsignedInt]
            4294967295 (0xffffffff) = Custom, 0 = QCIF, 1 = CIF, 2 = 2CIF, 3 = 4CIF, 4 = D1. If set to Custom (-1), the customResHeight and customResWidth tags must also be specified.
          • tvformat [unsignedInt]
            Resolution of the stream. Can be one of the following:
            • -2 for an undefined resolution
              The undefined resolution value is currently only valid for our Sarix product line.
            • -1 for a custom resolution
              The custom resolution value is currently only valid for our Sarix product line. For a list of valid Sarix resolutions by camera model, please refer to the Sarix Video Specifications section. If selected, customResHeight and customResWidth must be specified.
            • 0 for QCIF (QSIF)
            • 1 for CIF (SIF)
            • 2 for 2CIF (2SIF)
            • 3 for 4CIF (4SIF)
            • 4 for D1
          • customResHeight - optional; [unsignedInt]
            Vertical value for a custom resolution
            customResHeight is currently only valid for our Sarix product line. For a list of valid Sarix resolution heights by camera model, please refer to the Sarix Video Specifications section. This must be specified if videosize is set to 'custom resolution' (-1).
          • customResWidth - optional; [unsignedInt]
            Horizontal value for a custom resolution
            CustomResWidth is currently only valid for our Sarix product line. For a list of valid Sarix resolution widths by camera model, please refer to the Sarix Video Specifications section. This must be specified if videosize is set to 'custom resolution' (-1).
          • gop [decimal]
            The number of frames in one GOP (Group of Pictures). For common values please refer to CameraConfiguration's video configuration table.
            Pelco streams do not adhere to the standard GOP structure. Streams will always have one intracoded frame (I frame), which resides in the beginning of the GOP, and then followed by predicted frames (P frames). They do not have bi-directional frames (B frames). Therefore if the gop value is 15, there will be one I-frame and 14 B-frames.
      • asyncId - optional; [int]
        This is only relevant to Endura and is related to multicast messages from GUI client software

    2. Disconnect

      Terminates the streaming session indicated by sessionId.
      SOAP action:
      urn:schemas-pelco-com:service:VideoOutput:1#Disconnect
      Input:
      [Disconnect]
      Represents a request to the Disconnect action
      • sessionId [string]
        Id of the session to disconnect from
      • asyncId - optional; [int]
        This is only relevant to Endura and is related to multicast messages from GUI client software
      Output:
      [DisconnectResponse]
      Represents a response from the Disconnect action
      • asyncId - optional; [int]
        This is only relevant to Endura and is related to multicast messages from GUI client software

    3. GetActiveSessionIds

      Returns a list of sessionIds for currently active sessions associated with the service instance.
      SOAP action:
      urn:schemas-pelco-com:service:VideoOutput:1#GetActiveSessionIds
      Input:
      [GetActiveSessionIds]
      Represents a request to get the Ids of active sessions
      Output:
      [GetActiveSessionIdsResponse]
      Represents a response from GetActiveSessionIds request
      • sessionIds [Sessions]
        List of active session Ids
        • session - optional, unbounded; [string]
          Represents a Session Id

    4. GetActiveSessions

      Returns the number of active sessions currently managed by the service instance as well as the maximum number of simultaneous sessions supported by the service instance.
      SOAP action:
      urn:schemas-pelco-com:service:VideoOutput:1#GetActiveSessions
      Input:
      [GetActiveSessions]
      Represents a request to get all the current active sessions
      Output:
      [GetActiveSessionsResponse]
      Represents a response from the GetActiveSessions request
      • activeSessions [int]
        The number of currently active sessions
      • maxSessions [int]
        The maximum number of live sessions supported

    6. Query

      Returns a catalog listing of video data managed by the service instance that is available for streaming and matches the search criteria specified by streamQuery.
      SOAP action:
      urn:schemas-pelco-com:service:VideoOutput:1#Query
      Input:
      [Query]
      Represents request to the Query Action
      • streamQuery [StreamQuery]
        Represents a query. StreamQuery only needs to contain elements needed to filter the search results; these elements are also used to specify which elements should be included in the response (this is akin to SQL's SELECT). The elements that should be included in the response should contain a "select" attribute marked 'true'. If a "select" attribute is associated with the root streamQuery element, all sub-elements will be returned.
        A direct transformation exists between the search criteria and an SQL statement of the form SELECT [syseng:Fields] FROM [syseng:Table] WHERE [syseng:Conditions] where [syseng:Fields] includes those elements with an associated "select" attribute (or is considered to be * if the streamQuery element has a "select" attribute), [syseng:Table] is empty, and [syseng:Conditions] is the logical and of the included search criteria.
        • pageHeader - optional; [PageHeader]
          Represents the paging data
          • queryId - optional; [QueryId]
            Unique Identifier when making querying the NSM/NVR
          • nextEntry - optional; [unsignedInt]
            The index where the next page begins within the result set
          • totalEntries - optional; [unsignedInt]
            The total number of entries returned for a particular response
          • maxEntries - optional; [unsignedInt]
            The maximum number of results being returned for the query
          • resultURI - optional; [string]
            The URL from which the search results can be downloaded
          • truncated - optional; [boolean]
            When set to true it would mean that even though the results returned are less than the max entries, there are more results available for the query. This element is only applicable to the recorders
        • sourceURI - optional; [SelectableString]
          Represents the source URI.
        • outputURI - optional; [SelectableString]
          Represents the output URI
        • eventURI - optional; [SelectableString]
          Represents the event URI
        • eventURN - optional; [SelectableString]
          Represents the event URN.
        • serviceId - optional; [string]
          Represents the unique identifier for the service
        • formats - optional; [FormatList]
          Represents the list of formats. The following are the valid values
          • Mpeg-4
          • H.264
          The formats are only valid for Sarix
          • format - optional; [string]
            Name of the format
        • resolutions - optional; [ResolutionList]
          A list of resolutions to be used in the search criteria. May be one of:
          • D1
          • 4CIF
          • 2CIF
          • CIF
          • QCIF
          • resolution - optional; [string]
            Name of the resolution
        • framerate - optional; [SelectableRange]
          Represents the frame rate
          • minimum [int]
            The starting value of the range of valid values.
          • maximum [int]
            The ending value of the range of valid values.
        • bitrate - optional; [SelectableRange]
          Represents the bit rate
          • minimum [int]
            The starting value of the range of valid values.
          • maximum [int]
            The ending value of the range of valid values.
        • startTime - optional; [SelectableString]
          The start time of the interval being played back.
          • The format of start time is dateTime.tz. ex 1988-04-07T18:39:09-08:00
          • UTC timezone
          • NOW is also valid
        • endTime - optional; [SelectableString]
          The end time of the interval being played back.
          • The format of the end time is dateTime.tz. ex 1988-04-07T18:39:09-08:00
          • UTC timezone
          • INFINITE is also valid for signifying no end time
        • deletionPriority - optional; [int]
          Represents the likelihood an NVR delete a particular stream when it needs to regain a certain amount of free disk space. This is restricted to the following values (lowest priority to highest - excluding UNDEFINED):
          • 0 for DISCARD
          • 1 for LOW
          • 2 for MEDIUM
          • 3 for HIGH
          • 4 for LOCK
          • 5 for UNDEFINED
      • asyncId - optional; [int]
        This is only relevant to Endura and is related to multicast messages from GUI client software
      Output:
      [QueryResponse]
      Represents the reponse from the Query action call
      • streamCatalog [StreamCatalog]
        Represents catalog of streams. It primarily consist of the paging data and the list of entries associated with the submitted search criteria.
        • pageHeader - optional; [PageHeader]
          The paging data
          • queryId - optional; [QueryId]
            Unique Identifier when making querying the NSM/NVR
          • nextEntry - optional; [unsignedInt]
            The index where the next page begins within the result set
          • totalEntries - optional; [unsignedInt]
            The total number of entries returned for a particular response
          • maxEntries - optional; [unsignedInt]
            The maximum number of results being returned for the query
          • resultURI - optional; [string]
            The URL from which the search results can be downloaded
          • truncated - optional; [boolean]
            When set to true it would mean that even though the results returned are less than the max entries, there are more results available for the query. This element is only applicable to the recorders
        • entries - optional; [Entries]
          List of stream parameters associated with the submitted search criteria
          • streamParameters - optional, unbounded; [StreamParameters]
            Stream Parameters List
            • displayName - optional; [string]
              The Stream's friendly name
            • eventType - optional; [string]
              Recording or event type that caused the stream to be recorded.The following are the valid values
              • Scheduled
              • Motion
              • Alarm
              • Manual
              Primarily only returned by NVR/NSM queries.
