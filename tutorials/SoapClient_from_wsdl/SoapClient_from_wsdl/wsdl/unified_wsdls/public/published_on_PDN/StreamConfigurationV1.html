StreamConfiguration - Web Service


$Id: StreamConfigurationV1.wsdl 433 2010-11-23 01:18:25Z $

This service is used to configure the properties of streams available from a device.

In order to make this service flexible and applicable to different devices, configuration settings are not explicitly specified. Instead, this service will return the available settings and their possible values. To specify a device's configuration, you need to define one setting at a time. This iterative approach is required because each setting choice can affect the possible values and availability for other settings.

Target Namespace:
urn:schemas-pelco-com:service:StreamConfiguration:1
[toc]

Operations

    1. ApplyDynamicSettings

      This operation enables setting of various parameters, e.g. the bit rate and frame rate, temporarily and dynamically without recalculating resources. This function will only be allowed to use a subset of the already identified maximum in order to reserve/allocate enough space without resetting the stream. It will be tied specifically to single video stream.
      SOAP action:
      urn:schemas-pelco-com:service:StreamConfiguration:1#ApplyDynamicSettings
      Input:
      [ApplyDynamicSettings]
      Represents a request to temporarily and dynamically apply one or more settings without restarting the stream.
      • streamType [string]
        describes the stream (audio, video, etc.)
      • streamSettingsList [StreamIDSettingsList]
        a StreamIDSettingsList data element
        • streamIDSettings - optional, unbounded; [StreamIDSettings]
          A list of StreamIDSettings data elements
          • stream [StreamID]
            A StreamID data element
            • type [string]
              The type of stream.
            • id [int]
              An integer value assigned by order of priority. The first stream is assigned the id 0. Subsequent streams are assigned consecutive values (1, 2, ...) in order of priority. 0 is the "primary" stream, 1 is the "secondary" stream, and so on.
          • settings [StreamSettings]
            A StreamSettings data element
            • setting - optional, unbounded; [Setting]
              A list of Setting data elements.
              • name [string]
                The name of the setting.
                • xsd:choice
                • decimalValue [decimal]
                  A decimal value used for specifying setting values such as framerate.
                • stringValue [string]
                  A string value used for specifying setting values such as encoding.
                • resolutionValue [Resolution]
                  A Resolution data element used for specifying resolution values.
                  • width [int]
                    The horizontal size of the image.
                  • height [int]
                    The vertical size of the image.
                • intValue [int]
                  An integer value used for specifying setting values such as gop, bitrate, and quality.
                • booleanValue [boolean]
                  A boolean value used for flags.
      Output:
      [ApplyDynamicSettingsResponse]
      Represents a response from the ApplyDynamicSettings action

    11. SaveSettings

      This action is now deprecated. Please use the SaveAllSettings action instead.

      This is the only operation that actually modifies the state of the camera. Once a configuration has been constructed, it can be passed to this operation to change the actual stream configuration on the camera. Users must pass in a full configuration and NOT a partial configuration, which includes all of the settings and their respective values; even for settings that are not being changed. Passing in only a partial configuration will result in an error.

      If the configuration is invalid or missing settings, it is rejected and an error is returned without modifying the state of the camera. Please refer to the SaveSettings in the StreamConfiguration Issues section for details.
