AlarmArrayConfiguration - Web Service


$Id: AlarmArrayConfigurationV1.wsdl 433 2010-11-23 01:18:25Z $
The AlarmArrayConfiguration service allows multiple alarms to be evented from a single service. Each alarm in the service is individually identified by an alarmID, which in essence is an index into the array. Each AlarmArrayConfiguration service should be wrapped up by an AlarmArray device, much like an AlarmControl service is wrapped up by an Alarm device.
Target Namespace:
urn:schemas-pelco-com:service:AlarmArrayConfiguration:1

Operations

    1. GetAlarmStates

      DEPRECATED: This action is now deprecated. It is not recommended for production usage. The recommended method is now using the Event Arbiter Library to subscribe to this service's events and alarms OR subscribing directly to the GENA events.

      Returns the current state of the all alarms in the service in bitmapped string.

      SOAP action:
      urn:schemas-pelco-com:service:AlarmArrayConfiguration:1#GetAlarmStates
      Input:
      [GetAlarmStates]
      Represents a request to the GetAlarmStates action.
      Output:
      [GetAlarmStatesResponse]
      Represents a response from the GetAlarmStates action.
      • alarmStates [AlarmStates]
        • objId - optional; [int]
        • offset - optional; [int]
          Zero based offset of the first alarm contained in the contained bitmasks. int 32. The range is 0 to (# of alarms - 1).
        • length [int]
          Number of relays states contained by the contained bitmasks. The range is 1 to (# of alarms)
        • changed - optional, nillable; [string]
          Bitmask of the change-of-state status of alarm contained within the array range defined by offset and length. The bitmask is in Pelco's base64 format. Pelco base64 is identical to defacto base64 encoding, except that Pelco base64 always appends an equal sign (=) at the end.
        • state1 [string]

          Bitmask of the first state bit for alarms contained within the range of the bitmask. The bitmask is in Pelco's base64 format. Pelco base64 is identical to defacto base64 encoding, except that Pelco base64 always appends an equal sign (=) at the end. Currently defined values are:

          • 0 for INACTIVE
          • 1 for ACTIVE
          • 2 for FAULT
          • 3 for SHORT

        • state2 [string]

          Bitmask of the second state bit for alarms contained within the range of the bitmask. The bitmask is in Pelco's base64 format. Pelco base64 is identical to defacto base64 encoding, except that Pelco base64 always appends an equal sign (=) at the end. Currently defined values are:

          • 0 for INACTIVE
          • 1 for ACTIVE
          • 2 for FAULT
          • 3 for SHORT

        • enabled [string]
          Bitmask of enabled states of alarm contained within the range of the bitmask. Pelco base64 is identical to defacto base64 encoding, except that Pelco base64 always appends an equal sign (=) at the end. Live data will be in Pelco's base64.

    3. GetConfiguration

      Get the configuration of the alarm input.
      SOAP action:
      urn:schemas-pelco-com:service:AlarmArrayConfiguration:1#GetConfiguration
      Input:
      [GetConfiguration]
      Respresents a request to the GetConfiguration action .
      • alarmID [int]
        An alarm's index location within an alarm array
      Output:
      [GetConfigurationResponse]
      Respresents a response from the GetConfiguration action.
      • alarmConfig [AlarmConfig]
        Represents an AlarmConfig data element.
        • severity - optional; [int]

          The level of importance of the alarm output.

          Within Endura systems, this value is ignored. Instead the System Manager completely handles alarm severity.

          This value does not correspond to severity values entered by the user through the workstation. Workstation entered severity values are only stored on System Managers, and do not update the severity values in the AlarmArray device itself.

        • polarity [int]
          The polarity of the alarm output. Valid values are:
          • 0 for NORMAL
          • 1 for REVERSE
        • enabled [int]
          The active state of the alarm output. Valid values are:
          • 0 for DISABLED (An alarm output that is disabled will not event the alarm state in response to a state change.)
          • 1 for ENABLED
        • supervised [int]

          Determines whether or not the alarm is operating in supervised mode. More specifically it determines whether or not the alarm input should be driven such that a loss of voltage to the input results in an alarm condition.

          • 0 for unsupervised
          • 1 for supervised

        • dwellTime [double]
          The time, in 100s milliseconds, that the alarm input should be reported as active when triggered, regardless of the actual state of the physical input. The value must be between 0 to 0.5.
        • physicalInput [int]

          An integer representing the physical alarm input with which this service is associated. Valid values:

          • 0 - indicates that the service maps to a physical alarm input
          • -1 - indicates that the service instance does not map to a physical alarm input. The alarm is software based. (e.g. Sarix returns this value, as they are software based).

        • combinedEventEntry - optional; [COMBINED_EVENT_ENTRY]
          A COMBINED_EVENT_ENTRY data element
          • opStack [OPSTACK]
            Represents an OPSTACK data element.
            • operator [string]

              The operator needed for a combined event. Valid values:

              • AND - expects two arguments
              • OR - expects two arguments
              • NOT - expects one argument
              • NOP - expects one argument

            • event_entries [EVENT_ENTRIES]
              Represents an EventEntries data element
              • eventEntry - unbounded; [EVENT_ENTRY]
                • uid [string]
                  The event's unique identifier.
                • eventURI [string]
                  The event's URI.
                • eventType [string]
                  The event's Type.
                • serviceType [string]
                  A service type URN.
                • propertySet [PROPERTY_SET]
                  • property - unbounded; [PROPERTY]
                    Represents a PROPERTY data element
                    • name [string]
                      The name of the property of the value to compare against. 250 max characters.
                    • value [string]
                      The value to compare against. i.e. The event is considered "active" when the current value of each listed property, compared, via the comparator, to the defined value yields true.
                    • comparator [string]
                      The comparison operator to utilize.

    4. ResetConfiguration

      Returns the alarm input to the default configuration state.
